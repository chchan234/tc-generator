# TC Generator - 자동 테스트 케이스 생성기

기획서 문서(PDF, DOCX)를 분석하여 상세 테스트 케이스(TC)를 자동으로 생성하고, LangGraph 기반 Multi-Agent 시스템으로 검수 및 피드백을 제공하는 도구입니다.

## 주요 기능

- **기획서 문서 업로드**: PDF, DOCX 형식의 기획서 파일 업로드 지원
- **텍스트 추출 및 전처리**: 문서에서 텍스트 추출 및 불필요한 내용 필터링
- **Chunking 및 요약**: 긴 문서의 경우 적절한 크기로 나누고 요약
- **테스트 케이스 자동 생성**: Google Gemini를 활용한 상세 TC 자동 생성
- **테스트 케이스 자동 검수**: LangGraph Multi-Agent 기반 TC 품질 검수 및 피드백
- **결과 내보내기**: 생성된 TC를 CSV, Excel 형식으로 저장 지원
- **기능 영역별 생성**: 기능 영역을 분석하여 더 상세한 테스트 케이스 생성

## 설치 방법

### 1. 저장소 클론 또는 다운로드

```bash
git clone https://github.com/chchan234/tc-generator.git
cd tc-generator
```

### 2. 필요한 패키지 설치

```bash
pip install -r requirements.txt
```

### 3. Google Gemini API 키 준비

- [Google AI Studio](https://makersuite.google.com/app/apikey)에서 API 키 발급
- 발급 받은 API 키는 애플리케이션 내에서 설정할 수 있습니다

## 상세 사용 방법

### 1. 애플리케이션 실행

```bash
python app.py
```

### 2. API 키 설정

![API 키 설정](https://github.com/chchan234/tc-generator/raw/main/docs/img/api_key_setting.png)

- 애플리케이션 우측 상단의 '설정' 버튼을 클릭합니다
- Google Gemini API 키를 입력하고 '저장' 버튼을 클릭합니다
- API 키는 애플리케이션 내에서만 임시로 저장되며, 종료 시 삭제됩니다

### 3. 기획서 파일 업로드

![파일 업로드](https://github.com/chchan234/tc-generator/raw/main/docs/img/file_upload.png)

- '파일 선택' 버튼을 클릭하여 PDF 또는 DOCX 형식의 기획서 파일을 선택합니다
- 파일이 선택되면 파일 경로가 표시됩니다
- 원활한 처리를 위해 50MB 이하의 파일을 권장합니다

### 4. 설정 구성

![설정 구성](https://github.com/chchan234/tc-generator/raw/main/docs/img/settings.png)

#### 4.1 테스트 케이스 템플릿 유형 선택
- **기본 템플릿**: 일반적인 기능 테스트를 위한 기본 형식
- **API 템플릿**: REST API 테스트에 특화된 형식
- **UI 템플릿**: 사용자 인터페이스 테스트에 특화된 형식
- **모바일 템플릿**: 모바일 앱 테스트에 특화된 형식

#### 4.2 청크 크기 및 중복 설정
- **청크 크기**: 4,000~8,000 사이 권장 (단위: 문자)
- **중복 크기**: 100~300 사이 권장 (단위: 문자)
- 문서가 너무 크면 청크 크기를 줄이고, 문맥 유지를 위해 중복 크기를 조절합니다

#### 4.3 테스트 케이스 자동 검수 옵션
- **검수 활성화**: 생성된 테스트 케이스의 품질을 자동으로 검수하고 피드백 제공
- 검수 과정에서 누락된 예외 사항, 경계값 테스트 등을 보완합니다

#### 4.4 생성 방식 선택
- **일반 생성**: 전체 문서를 기반으로 테스트 케이스를 한 번에 생성
- **기능 영역별 생성**: 문서에서 주요 기능 영역을 자동 식별하고 각 영역별로 세분화된 테스트 케이스 생성
  - 영역별 생성은 더 상세하고 체계적인 테스트 케이스 생성에 효과적입니다
  - 대규모 프로젝트에 권장되는 방식입니다

### 5. 처리 시작

![처리 시작](https://github.com/chchan234/tc-generator/raw/main/docs/img/processing.png)

- 모든 설정을 완료한 후 "처리 시작" 버튼을 클릭합니다
- 진행 상황이 진행 표시줄과 상태 메시지로 표시됩니다
- 문서 크기와 설정에 따라 처리 시간이 달라질 수 있습니다:
  - 5페이지 내외의 소규모 문서: 약 1-2분
  - 20페이지 내외의 중간 규모 문서: 약 5-10분
  - 50페이지 이상의 대규모 문서: 10분 이상

### 6. 결과 확인 및 내보내기

![결과 확인](https://github.com/chchan234/tc-generator/raw/main/docs/img/results.png)

- 처리가 완료되면 탭을 통해 다양한 결과를 확인할 수 있습니다:
  - **원본 텍스트**: 문서에서 추출된 원본 텍스트
  - **처리된 텍스트**: 전처리 및 청킹된 텍스트
  - **테스트 케이스**: 자동 생성된 테스트 케이스 목록
  - **피드백**: 테스트 케이스 품질에 대한 자동 검수 피드백
  
- 테스트 케이스와 피드백 결과는 다음과 같은 형식으로 내보낼 수 있습니다:
  - **CSV 내보내기**: 간단한 텍스트 편집기나 스프레드시트 프로그램에서 열기 좋은 형식
  - **Excel 내보내기**: 서식 및 필터링 기능이 포함된 형식

### 7. 결과물 활용

![결과물 활용](https://github.com/chchan234/tc-generator/raw/main/docs/img/excel_output.png)

- 생성된 Excel 파일에는 다음 내용이 포함됩니다:
  - **ID**: 테스트 케이스 고유 번호
  - **카테고리**: 테스트 영역 또는 기능 분류
  - **제목**: 테스트 케이스 요약
  - **선행 조건**: 테스트 실행을 위한 사전 조건
  - **테스트 단계**: 상세 테스트 실행 단계
  - **기대 결과**: 각 테스트 단계에서 기대되는 결과
  - **우선순위**: 테스트 케이스 중요도
  - **유형**: 테스트 케이스 유형(기능, 성능, 예외 등)
  - **자동화 가능**: 자동화 테스트 적합성 여부
  - **비고**: 추가 설명이나 참고사항

## 기능 영역별 생성 방식 활용하기

기능 영역별 생성 방식은 다음과 같이 동작합니다:

1. **기능 영역 식별**: 문서를 분석하여 주요 기능 영역(로그인, 회원가입, 검색 등)을 자동으로 식별
2. **영역별 템플릿 적용**: 각 기능 영역에 최적화된 템플릿 생성
3. **세부 테스트 케이스 생성**: 영역별로 더 상세한 테스트 케이스 생성
4. **결과 통합**: 모든 영역의 테스트 케이스를 체계적으로 통합

### 권장 사용 시나리오:

- **대규모 프로젝트**: 다양한 기능이 포함된 대규모 프로젝트
- **복잡한 시스템**: 다수의 상호작용 모듈이 포함된 시스템
- **단계적 개발 프로젝트**: 각 기능별로 순차적 개발이 진행되는 프로젝트

### 장점:

- **높은 상세도**: 각 기능 영역에 대해 더 깊이 있는 테스트 케이스 생성
- **체계적 구조**: 기능별로 구조화된 테스트 케이스 세트 생성
- **효율적 관리**: 기능별로 테스트 진행 상황 추적 용이
- **중복 감소**: 기능 간 경계가 명확하여 중복 테스트 감소

## 실행 파일 빌드

실행 파일(.exe)을 빌드하려면 다음 명령을 실행하세요:

```bash
python create_exe.py
```

빌드된 실행 파일은 `dist/tc_generator` 폴더에 생성됩니다.

실행 파일 빌드 시 주의사항:
- PyInstaller가 설치되어 있어야 합니다
- 빌드에는 컴퓨터 사양에 따라 5-10분 정도 소요됩니다
- `onedir` 방식으로 생성되어 압축 해제 시간이 필요하지 않습니다

## 문제 해결

### 자주 발생하는 문제와 해결 방법:

1. **API 키 인증 오류**
   - API 키가 올바르게 입력되었는지 확인
   - Google Cloud Platform에서 API 키의 활성화 상태 확인
   - 인터넷 연결 상태 확인

2. **문서 파싱 오류**
   - 지원되는 파일 형식(PDF, DOCX)인지 확인
   - 파일이 손상되었는지 확인
   - 복잡한 서식의 경우 텍스트로 변환 후 시도

3. **메모리 부족 오류**
   - 청크 크기를 줄여 메모리 사용량 감소
   - 대용량 문서의 경우 여러 부분으로 나누어 처리
   - 다른 실행 중인 프로그램을 종료하여 메모리 확보

4. **처리 시간 과도 문제**
   - 청크 크기 조정
   - 더 작은 문서로 분할하여 처리
   - 불필요한 섹션을 제외하고 처리

5. **실행 파일 실행 오류**
   - 안티바이러스 프로그램에서 예외 등록
   - 관리자 권한으로 실행
   - 모든 필요한 리소스 파일이 올바른 위치에 있는지 확인

## 시스템 요구사항

- **운영체제**: Windows 10/11, macOS 10.14 이상, Linux (Ubuntu 18.04 이상)
- **메모리**: 최소 4GB RAM (8GB 이상 권장)
- **저장공간**: 최소 500MB 여유 공간
- **프로세서**: 듀얼 코어 이상 (쿼드 코어 이상 권장)
- **인터넷 연결**: 필수 (Google Gemini API 사용)
- **Python**: 3.9 이상 (독립 실행 파일 사용 시 불필요)

## 프로젝트 구조

```
tc-generator/
├── app.py                 # 메인 애플리케이션
├── create_exe.py          # 실행 파일 빌드 스크립트
├── src/
│   ├── document_processor.py  # 문서 처리
│   ├── text_extractor.py      # 텍스트 추출
│   ├── chunker.py             # 청킹 및 요약
│   ├── tc_generator.py        # TC 생성 (Gemini 연동)
│   ├── tc_reviewer.py         # LangGraph 활용 검수
│   └── file_manager.py        # 파일 저장 관리
├── output/                # 결과물 저장 디렉토리
├── templates/             # GUI 템플릿
├── static/                # 정적 파일
└── requirements.txt       # 의존성 패키지
```

## 주의사항

- API 키는 안전하게 관리해주세요. 애플리케이션 내에서는 일시적으로만 저장됩니다.

- 실행 파일(.exe)은 처음 실행 시 압축 해제에 시간이 걸릴 수 있습니다.
